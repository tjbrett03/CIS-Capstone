CIS Capstone — Story Interview Application

Setup, Configuration, and User Guide

Overview

This application helps nonprofit staff surface powerful stories from their program work. A staff member selects a content goal and a target audience, then answers a short guided interview conducted by a local language model (Ollama). Once the interview is complete, the extracted story is sent to Claude (Anthropic API) to generate a finished, audience-ready narrative.

The output screen displays the narrative with any sensitive information flagged for review and a readability score compared against the target audience level.

Architecture

  • Flask — Python web framework, handles routing, sessions, and auth
  • Ollama — local LLM runtime; conducts the interview and extracts story elements as JSON
  • Anthropic API (Claude) — generates the finished narrative from the extracted JSON
  • Presidio + spaCy — PII scanning on the generated narrative
  • textstat — Flesch Reading Ease readability scoring
  • Docker Compose — runs the Flask app and Ollama together as containers

Prerequisites

Running with Docker (recommended)

  • Docker Desktop with the WSL2 backend enabled (Windows) or Docker Engine (Linux)
  • NVIDIA drivers if using GPU passthrough (optional but recommended for speed)

Running locally without Docker

  • Python 3.12 or later
  • Ollama installed and running locally (ollama.com)

Setup

1. Clone the repository

git clone https://github.com/tjbrett03/CIS-Capstone.git
cd CIS-Capstone

2. Configure environment variables

Copy the example environment file and fill in the required values:

cp .env.example .env

Running with Docker

1. Start the stack

docker compose up --build -d

This builds the Flask app image (including the spaCy language model) and starts both the app and Ollama containers. The build takes a few minutes on first run due to the spaCy model download.

2. Pull the language model

On first run, pull the Ollama model into the container. This only needs to be done once — the model is stored in a named Docker volume and persists across restarts.

docker compose exec ollama ollama pull llama3.1:8b

3. Open the application

Visit http://localhost:5000 in your browser.

GPU passthrough: GPU support is enabled by default in docker-compose.yml for NVIDIA GPUs. To run CPU-only, comment out or remove the deploy block in docker-compose.yml.

Running Locally (without Docker)

1. Create and activate a virtual environment

python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

2. Install dependencies

pip install -r requirements.txt

3. Download the spaCy language model

python -m spacy download en_core_web_lg

4. Start Ollama and pull a model

ollama pull llama3.1:8b

5. Run the application

python run.py

Visit http://localhost:5000 in your browser.

Using the Application

Step 1 — Sign in

Enter the password configured in APP_PASSWORD.

Step 2 — Select a goal and audience

Choose a content goal (what the narrative will be used for) and a target audience (who will read it). These two selections shape every part of the interview and the final narrative.

Step 3 — Complete the interview

The local language model conducts a structured five-question interview, collecting five story elements in order:

  1. Person — who is at the center of this story
  2. Moment — a specific day or conversation where something shifted
  3. Tension — what the person was up against before the change
  4. Change — what is different now compared to where they started
  5. Outcome — what this means for them going forward

Answer each question in your own words. The model may ask one or two follow-up questions if more detail would help. Type your response and press Enter or click Send.

Step 4 — Review the narrative

Once all five elements are collected, the model extracts them and sends the story to Claude, which generates a finished narrative tailored to the selected goal and audience. The narrative appears in the chat window.

Step 5 — Review PII flags

The output screen scans the narrative for six categories of personally identifiable information:

  1. Direct identifiers — names, initials, nicknames
  2. Location identifiers — specific facilities, shelters, treatment centers
  3. Relationship identifiers — names of family members (e.g. "her daughter Sofia")
  4. Case or legal identifiers — court names, case numbers, legal proceedings with dates
  5. Health or behavioral identifiers — specific diagnoses, medication names
  6. Immigration or legal status — references to undocumented status, deportation, parole

Any flagged content is highlighted in amber within the narrative. A panel below lists each flag with a plain-language explanation of why it was flagged and a suggested fix. The system warns but does not block — the staff member is the final decision-maker before sharing.

Step 6 — Check the readability score

A Flesch Reading Ease score is calculated for the narrative and compared against the target reading level for the selected audience. The result is displayed as a plain-language grade label (e.g. "7th–8th grade") with a note indicating whether it matches, exceeds, or falls below the target.

Step 7 — Copy and use

Click Copy narrative to copy the plain text to the clipboard. Paste it into any document, email, or CMS. Use Start Over to begin a new interview with the same goal and audience, or return to the home screen to change either selection.

Stopping and Restarting

# Stop the stack
docker compose down

# Restart without rebuilding (model and sessions are preserved)
docker compose up -d

# Rebuild after code changes
docker compose up --build -d web
Note: Interview data and session files are stored in the interviews/ and flask_session/ directories, which are mounted as volumes in Docker. They persist across container restarts. The Ollama model is stored in a named Docker volume (ollama_data) and also persists.
